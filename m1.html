Azure AI Services — Plan & Manage

What are Azure AI Services?

Azure AI Services (formerly Cognitive Services) are cloud-based REST APIs and SDKs that allow you to add AI capabilities to applications without needing ML expertise. They are organised into categories: Vision, Speech, Language, Decision, and Azure OpenAI.

Single-Service Resource
  • One endpoint per service (e.g., Speech only)
  • Dedicated billing & quota per service
  • Use when you need one specific AI service
Multi-Service Resource
  • Single endpoint for multiple services
  • Single key authenticates all services
  • Consolidated billing — easier to manage
  • Resource type: CognitiveServices

Authentication

Azure AI Services support two primary authentication methods:

Subscription Key (API Key)

Passed as Ocp-Apim-Subscription-Key header. Two keys provided — rotate without downtime.

Microsoft Entra ID (AAD)

Use service principals or managed identities. Preferred for production — no secrets in code.

Token-Based Auth

Exchange key for a 10-minute JWT token at /sts/v1.0/issueToken. Pass in Authorization: Bearer {token}.

// Authenticate with API Key
var client = new TextAnalyticsClient(
    new Uri("https://<endpoint>.cognitiveservices.azure.com"),
    new AzureKeyCredential("<api-key>")
);

// Authenticate with Managed Identity
var credential = new DefaultAzureCredential();
var client2 = new TextAnalyticsClient(endpoint, credential);

Securing AI Services

  • Azure Key Vault — store API keys; grant access via managed identities, never hardcode secrets
  • Network restrictions — limit access by virtual network or IP allowlists in the resource Networking tab
  • Private Endpoints — traffic stays within Azure backbone; disable public access
  • RBAC — assign Cognitive Services User to allow API calls; Cognitive Services Contributor to manage the resource
  • Diagnostic logging — send to Log Analytics; query with KQL
Best Practice Always use Managed Identity + Key Vault in production. For local dev, use DefaultAzureCredential which automatically picks up az login credentials.

Deploying in Containers

Many AI Services can run in Docker containers for on-premises or edge scenarios.

Key Facts Even in containers, the service must phone home to Azure for billing. Containers require Billing and ApiKey environment variables pointing to your Azure resource. The model is bundled in the container image — inference runs locally. 
# Pull and run a Language Detection container
docker pull mcr.microsoft.com/azure-cognitive-services/textanalytics/language:latest

docker run --rm -it -p 5000:5000 \
  -e ApiKey="<key>" \
  -e Billing="https://<resource>.cognitiveservices.azure.com/" \
  -e Eula=accept \
  mcr.microsoft.com/azure-cognitive-services/textanalytics/language:latest
