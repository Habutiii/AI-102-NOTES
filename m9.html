Content Safety & Responsible AI

Azure AI Content Safety

Azure AI Content Safety detects harmful content in both text and images, returning a severity score for each harm category.

Harm Categories

CategoryDescriptionSeverity Scale
HateDiscrimination or derogatory content targeting identity groups0, 2, 4, 6 (safe → severe)
ViolencePhysical harm, weapons, gore0, 2, 4, 6
SexualSexually explicit or suggestive content0, 2, 4, 6
Self-HarmSuicide, self-injury, eating disorders0, 2, 4, 6
Severity levels 0 = safe. 2 = low. 4 = medium. 6 = high/severe. You configure thresholds per category to decide when to block content. Threshold of 2 means any score ≥ 2 is rejected.

Content Safety API — Text Analysis

POST https://<resource>.cognitiveservices.azure.com/contentsafety/text:analyze?api-version=2024-09-01

{
  "text": "User message to analyze",
  "categories": ["Hate", "Violence", "Sexual", "SelfHarm"],
  "outputType": "FourSeverityLevels"
}

Advanced Content Safety Features

Prompt Shields

Detects two types of attacks: Direct jailbreaks (user tries to override system instructions) and Indirect prompt injection (malicious instructions embedded in documents or web pages fed to the model).

Groundedness Detection

Checks whether a model's response is factually supported by the supplied context/sources. Flags hallucinations where the model makes claims not found in the grounding documents.

Protected Material Detection

Identifies copyrighted text (e.g., song lyrics, book passages, code with restrictive licences) in model outputs to help avoid IP infringement.

Custom Blocklists

Add domain-specific terms that must always be blocked regardless of severity score. Applied on top of built-in harm detection.

Content Filtering in Azure OpenAI

Azure OpenAI includes a default content filter applied to all inputs and outputs. It uses the same four harm categories with configurable severity thresholds.

Filter PolicyDefault ThresholdDescription
HateMedium (≥ 4 blocked)Applies to both prompt and completion
ViolenceMediumApplies to both prompt and completion
SexualMediumApplies to both prompt and completion
Self-HarmMediumApplies to both prompt and completion
Custom filter policies You can create custom content filter configurations in Azure AI Foundry to lower or raise thresholds per category, or add blocklists. Requires applying to Microsoft for some threshold changes.

Responsible AI Principles (Microsoft)

Fairness

AI systems should treat all people fairly and not create or reinforce unfair bias. Monitor for disparate impact across demographic groups.

Reliability & Safety

AI must behave reliably under expected and unexpected conditions. It should fail gracefully without causing harm.

Privacy & Security

Protect personal data, comply with privacy regulations, and defend against adversarial inputs. Data used to train should be handled with appropriate consent.

Inclusiveness

AI should empower everyone, including people with disabilities. Accessible design and attention to underrepresented groups.

Transparency

People should know when they're interacting with AI. Model behaviour should be understandable and explainable where possible.

Accountability

Humans must remain accountable for AI decisions. Include human-in-the-loop oversight for high-stakes decisions.

Limited Access Features

The following AI capabilities require explicit Microsoft approval before use:

  • Face Identification & Verification (PersonGroup-based)
  • Face emotion detection attributes
  • Celebrity recognition in Video Indexer
  • Custom Neural Voice (TTS)
  • Speaker diarization for third-party call centres
