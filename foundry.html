Azure AI Foundry

Key Takeaways:
  • Unified platform for building, evaluating, and deploying AI applications
  • Access to foundation models (GPT-4, Llama) via model catalog and deployments
  • Build conversational applications, knowledge management, and agentic systems
  • Safety and quality evaluation tools to assess model performance and safety
  • Deploy to Azure App Service, Azure Container Instances, or Kubernetes

What is Azure AI Foundry?

Azure AI Foundry (formerly Azure AI Studio) is the unified platform for building, evaluating, and deploying enterprise-grade AI applications and agents. It combines model access, development tools, safety evaluation, and deployment into a single portal at ai.azure.com.

Key Concept Azure AI Foundry is the go-to platform for AI-102. It consolidates model catalog, prompt flow, evaluation, fine-tuning, and the Azure AI Foundry SDK. Many exam questions relate to choosing the right tool within Foundry.

Hub and Project Architecture

Foundry uses a two-level hierarchy:

Hub

Top-level resource. Defines shared infrastructure: connections to Azure services (OpenAI, Search, Storage), network policy, compute, and identity. Created once per team/organization.

Project

Team workspace within a Hub. Where you build, test, and deploy AI apps. Projects inherit Hub connections but can have their own resources. Used to organize work by app or team.

Browse and deploy models from multiple providers without managing infrastructure.

Exam Tip — PTU vs Standard PTU (Provisioned Throughput Units) gives guaranteed throughput and lower, consistent latency — choose it when you need predictable performance in production. Standard is cheaper for irregular workloads. PTU is billed hourly regardless of usage.

Prompt Flow

Prompt Flow is a visual LLM application builder inside Foundry. It lets you create, test, and evaluate multi-step AI pipelines.

DAG Flow

Directed Acyclic Graph of connected nodes. Each node is a step: LLM call, Python code, tool invocation, or prompt template. General-purpose.

Chat Flow

Optimised for multi-turn conversational applications. Includes chat history management and a built-in test interface.

Evaluation Flow

Tests another flow against a dataset. Returns quality and safety metrics for comparison. Used to benchmark prompt or model changes.

Flow Node Types

Fine-tuning in Azure AI Foundry

Fine-tune supported models using your own training data within the Foundry portal.

  • Supported models: GPT-4o, GPT-4o mini, GPT-3.5 Turbo, Phi-3
  • Data format: JSONL with {"messages": [{"role": "system", ...}, {"role": "user", ...}, {"role": "assistant", ...}]}
  • Process: Upload training data → Create fine-tuning job → Monitor → Evaluate → Deploy as new deployment
  • Validation data: Optional; used to measure overfitting
Fine-tuning vs RAG Prefer RAG for keeping responses grounded in up-to-date private data. Use fine-tuning only when you need consistent output style, format, or tone that prompt engineering alone cannot achieve.

Azure AI Foundry SDK

The azure-ai-projects SDK provides programmatic access to Foundry resources from code.

from azure.ai.projects import AIProjectClient
from azure.identity import DefaultAzureCredential

client = AIProjectClient.from_connection_string(
    conn_str="<project-connection-string>",
    credential=DefaultAzureCredential()
)

# Get an Azure OpenAI client scoped to this project
openai_client = client.inference.get_azure_openai_client(api_version="2024-12-01-preview")

# Retrieve a connection (e.g., Azure AI Search)
search_conn = client.connections.get("my-search-connection")

# Run an evaluation
eval_result = client.evaluations.create(
    evaluation={"display_name": "RAG eval run", ...}
)
