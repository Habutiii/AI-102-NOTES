Azure OpenAI Service & Generative AI

Key Takeaways:
  • Deploy GPT-4, GPT-3.5, Davinci, and Embedding models via Azure OpenAI Service
  • Access through named deployments with REST API and SDK
  • Use system and user messages for instruction tuning and few-shot prompting
  • Implement content filtering and responsible AI practices
  • Build RAG (Retrieval-Augmented Generation) systems with embeddings and semantic search

Azure OpenAI Service Overview

Azure OpenAI Service provides access to OpenAI models with Azure enterprise features: private networking, RBAC, compliance, content filtering, and responsible AI guardrails. Models are accessed through named deployments within an Azure OpenAI resource.

Resource → Deployment → Model You deploy a model to your resource as a named deployment (e.g., gpt-4o-prod) and then reference that deployment name in API calls — not the model name directly.

Chat Completions API

from openai import AzureOpenAI

client = AzureOpenAI(
    azure_endpoint="https://<resource>.openai.azure.com",
    api_key="<key>",
    api_version="2024-12-01-preview"
)

response = client.chat.completions.create(
    model="gpt-4o-prod",        # deployment name, not model name
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user",   "content": "Summarise Azure AI Search in 2 sentences."}
    ],
    temperature=0.7,
    max_tokens=500
)
print(response.choices[0].message.content)

Multimodal — GPT-4o Vision

GPT-4o accepts both text and images in the same request. Pass images as URLs or base64-encoded data.

response = client.chat.completions.create(
    model="gpt-4o-prod",
    messages=[{
        "role": "user",
        "content": [
            {"type": "text",      "text": "What does this diagram show?"},
            {"type": "image_url", "image_url": {"url": "https://example.com/arch.png"}}
        ]
    }]
)

Prompt Engineering

Zero-shot

Instruction only, no examples. Works for well-known tasks: "Classify this review as positive or negative."

Few-shot

Provide 2–5 input/output examples before the real query. Steers the model's output format and style.

Chain of Thought

Instruct the model to reason step-by-step. Add "Think step by step." Improves accuracy on multi-step reasoning tasks.

System Message

Sets the model's persona, tone, and constraints for the entire conversation. Applied before any user message.

Key Parameters
  • temperature (0–2): 0 = deterministic greedy. Higher = more creative/random. Use low values for factual tasks.
  • top_p (0–1): Nucleus sampling. Use either temperature or top_p, not both.
  • max_tokens: Max output tokens (input tokens not included).
  • frequency_penalty: Reduces repeated phrases.
  • presence_penalty: Increases topic diversity.
  • stop: One or more strings that stop generation when produced.

Retrieval-Augmented Generation (RAG)

RAG grounds model responses in your private/current data without fine-tuning. The model is restricted to answer only from retrieved context.

# RAG pattern
User query
  ↓
1. Embed query with text-embedding-3-large
2. Hybrid search Azure AI Search → top-K chunks
3. Build prompt:
     System: "Answer using only the sources below. If unsure, say so."
     Context: [retrieved chunks with source citations]
     User: [original query]
4. Call GPT-4o → grounded, cited answer
Azure OpenAI On Your Data

Built-in RAG that connects Azure OpenAI directly to Azure AI Search, Blob Storage, or Cosmos DB. No custom retrieval pipeline needed for simple scenarios.

Custom RAG Pipeline

Full control: embed documents → AI Search index → retrieval → prompt construction → GPT call. Use when you need custom chunking, hybrid search, or re-ranking.

Fine-tuning

Adapt a base model on domain-specific training examples (JSONL format). Supported models include GPT-4o mini and GPT-3.5 Turbo.

ApproachUse whenCost
Prompt engineeringStart here — works for most casesInference only
RAGAnswers need private or up-to-date knowledgeInference + search
Fine-tuningSpecific output style/format that prompting can't achieveTraining + higher inference
