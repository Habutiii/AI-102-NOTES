Azure AI Translator

Key Takeaways:
  • Neural machine translation supporting 100+ languages and scripts
  • Uses global endpoint (not region-specific) for translation
  • Transliterate text between different writing systems
  • Detect language of input text automatically
  • Support custom translation models via Document Translation API for large-scale workflows

Overview

Azure AI Translator is a cloud-based neural machine translation service supporting 100+ languages and scripts. It is distinct from the Azure AI Language service and uses a global endpoint.

Global Endpoint — Exam Trap Translator uses api.cognitive.microsofttranslator.com — a global, non-regional URL. When using an Azure AI Services multi-service resource key (formerly "Cognitive Services"), you must also send the Ocp-Apim-Subscription-Region header. Single-service Translator keys do not require this header.
Exam Tip — Three Core Translator Features (Q31) The three primary features of Azure AI Translator are:
  1. Text Translation — translate text between 100+ languages using neural machine translation
  2. Transliteration — convert text from one script to another (e.g., Japanese Hiragana → Latin). Changes the script/phonetics only — does NOT translate the meaning.
  3. Language Detection — identify which language a piece of text is written in, returning a language code and confidence score
Dictionary lookup, Dictionary examples, and Document Translation are additional features but not the core three.

Text Translation Features

Text Translation Request

# Translate to French and German simultaneously
POST https://api.cognitive.microsofttranslator.com/translate
    ?api-version=3.0
    &to=fr
    &to=de
    &from=en           # optional; auto-detected if omitted

Headers:
  Ocp-Apim-Subscription-Key: <key>
  Ocp-Apim-Subscription-Region: eastus    # required for multi-service key
  Content-Type: application/json

[ { "text": "Hello, world!" } ]
# Response
[{
  "detectedLanguage": { "language": "en", "score": 1.0 },
  "translations": [
    { "text": "Bonjour le monde!", "to": "fr" },
    { "text": "Hallo Welt!", "to": "de" }
  ]
}]

Transliteration Request

# Convert Japanese hiragana to Latin
POST /transliterate?api-version=3.0&language=ja&fromScript=jpan&toScript=latn

[ { "text": "こんにちは" } ]

Document Translation

Document Translation is an asynchronous batch operation that translates entire documents while preserving formatting and structure.

Supported Formats

PDF, DOCX, PPTX, XLSX, HTML, TXT, Markdown, RTF, XLIFF, CSV, and more.

Storage Requirement

Source and target documents must be in Azure Blob Storage containers. The service reads source and writes translated output to the target container.

Async Pattern

Submit a batch job → poll the operation URL → retrieve results when status is Succeeded.

# Submit a document translation batch
POST https://<resource>.cognitiveservices.azure.com/translator/text/batch/v1.1/batches

{
  "inputs": [{
    "source": {
      "sourceUrl": "https://<storage>.blob.core.windows.net/source?<SAS>",
      "language": "en"
    },
    "targets": [{
      "language": "fr",
      "targetUrl": "https://<storage>.blob.core.windows.net/target-fr?<SAS>"
    }]
  }]
}

Custom Translator

Train domain-specific translation models to improve accuracy for specialised terminology.

Training Data

Upload parallel corpora — matched source and target language document pairs. Minimum: 10,000 parallel sentences per language pair.

Using a Custom Model

Add the category query parameter with your custom model ID to the /translate endpoint. Falls back to generic model if custom model fails.

POST /translate?api-version=3.0&to=de&category=<customModelId>
[ { "text": "Apply torque to the crankshaft assembly." } ]

Python SDK

from azure.ai.translation.text import TextTranslationClient
from azure.core.credentials import AzureKeyCredential

client = TextTranslationClient(
    endpoint="https://api.cognitive.microsofttranslator.com",
    credential=AzureKeyCredential(key),
    region="eastus"
)

result = client.translate(
    body=["Hello, world!"],
    to_language=["fr", "ja"]
)

for translation in result[0].translations:
    print(f"{translation.to}: {translation.text}")
Exam Tips — Translator
  • Global endpoint — not regional like most Cognitive Services
  • Multi-service key: must send Ocp-Apim-Subscription-Region header
  • Document Translation uses Azure Blob Storage — not inline content
  • Custom Translator improves domain vocabulary; supply parallel corpora
  • Language auto-detection is built into the translate call (no separate step needed)
