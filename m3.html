Azure AI Language

Key Takeaways:
  • Sentiment analysis with opinion mining for fine-grained sentiment extraction
  • Entity recognition (NER) and entity linking to knowledge bases
  • Language detection and key phrase extraction
  • PII detection to identify personally identifiable information in text
  • Use Azure AI Translator for machine translation with custom models

Overview

Azure AI Language is the unified NLP service providing pre-built and custom language capabilities through a single endpoint. All features are accessed via the /language/:analyze-text REST endpoint or the azure-ai-textanalytics SDK.

Single endpoint, many features All pre-built features use the same base URL: POST /language/:analyze-text?api-version=2023-04-01 with a "kind" field in the body selecting the feature.

Pre-built Features

Sentiment Analysis

Document-level and sentence-level sentiment: positive, neutral, negative, mixed with confidence scores.

Opinion Mining

Aspect-based sentiment. Extracts target aspects and their opinions. E.g., "room: positive, service: negative". Enable with "opinionMining": true.

Key Phrase Extraction

Returns main talking points from text. No training needed. Useful for summarising large volumes of text.

Language Detection

Returns ISO 639-1 language code + confidence score. Returns (Unknown) for mixed-language or ambiguous text.

Named Entity Recognition (NER)

Extracts typed entities: Person, Location, Organization, Date, Quantity, URL, Email, IP, Phone, etc.

Entity Linking

Disambiguates entities by linking to Wikipedia. E.g., "Mars" → Mars (planet) vs. Mars (candy bar) with confidence score.

PII Detection

Identifies and optionally redacts Personally Identifiable Information: SSNs, credit card numbers, emails, phone numbers, addresses.

Text Summarization

Two modes: extractive (selects key sentences verbatim) and abstractive (generates new summary text). Both are async operations.

REST API — Sentiment Analysis

POST https://<endpoint>.cognitiveservices.azure.com/language/:analyze-text?api-version=2023-04-01

{
  "kind": "SentimentAnalysis",
  "analysisInput": {
    "documents": [
      { "id": "1", "language": "en", "text": "I love Azure AI services!" }
    ]
  }
}

Text Summarization

Both summarization types use an async long-running operation pattern (submit → poll → retrieve).

TypeHow It WorksOutput
ExtractiveScores each sentence by importance and selects the top N. Sentences are verbatim from the original text.List of sentences with rank score, offset, length
AbstractiveGenerates new summary text using a generative model. May not match exact phrases in the source.Generated summary paragraph(s)
from azure.ai.textanalytics import TextAnalyticsClient
from azure.core.credentials import AzureKeyCredential

client = TextAnalyticsClient(endpoint, AzureKeyCredential(key))

# Extractive summarization — returns key sentences verbatim
poller = client.begin_extract_summary(
    ["Long article text..."],
    max_sentence_count=3
)
for doc in poller.result():
    for sentence in doc.sentences:
        print(sentence.text, f"(rank: {sentence.rank_score:.2f})")

# Abstractive summarization — generates new summary text
poller = client.begin_abstract_summary(
    ["Long article text..."],
    sentence_count=2
)
for doc in poller.result():
    print(doc.summaries[0].text)

Custom Text Classification

Single-label Classification

Each document gets exactly one class. Use when categories are mutually exclusive (e.g., Sports / Politics / Technology).

Multi-label Classification

A document can belong to multiple classes. Use when overlap is possible (e.g., a film that is both Action and Comedy).

Workflow — Custom Features (Classification, NER, Q&A) Language Studio → Create project → Upload & label data → Train → Evaluate (precision / recall / F1) → Deploy → Call prediction endpoint

Custom Named Entity Recognition

Extend pre-built NER with domain-specific entity types. Examples: part numbers, medical procedure codes, legal clause references.

  • Minimum: ~10 labeled examples per entity type
  • Recommended: 50–200 examples for production accuracy
  • Labels are applied at the character span level in Language Studio

Question Answering

Builds a knowledge base (KB) from FAQs, documents, or URLs and answers natural language questions.

Custom Question Answering

Build KB from documents, URLs, or Q&A pairs. Supports multi-turn conversations (follow-up prompts, up to 5 per answer). Requires Azure AI Language resource; uses Azure AI Search for large KBs.

Pre-built Q&A

Query a passage of text directly — no KB, no training. One-shot extraction. Useful for reading comprehension tasks.

Exam Tip — Q&A vs CLU Q&A: FAQ-style knowledge retrieval ("What are your opening hours?"). CLU: Intent understanding to trigger app actions ("Book me a flight to Paris next Friday"). If the question has a document/KB answer → Q&A. If you need to extract slots and route to an action → CLU.
