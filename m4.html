Azure AI Speech

Key Takeaways:
  • Speech-to-Text (STT): Recognize speech with continuous and interactive modes
  • Text-to-Speech (TTS): Convert text to natural-sounding audio with multiple voices and styles
  • Speaker recognition for speaker identification and verification
  • Configure regional endpoints and manage authentication with API keys
  • Handle streaming audio input and output for real-time applications

Speech Service Capabilities

Speech-to-Text (STT)

Converts spoken audio to text. Supports real-time and batch transcription. Can use custom acoustic/language models.

Text-to-Speech (TTS)

Converts text to natural-sounding audio. 400+ voices, 140+ languages. Supports SSML for customization.

Speech Translation

Real-time speech translation — transcribes and translates simultaneously. Uses Speech SDK.

Speaker Recognition

Identifies or verifies speakers by voice. Speaker Verification (1:1) and Speaker Identification (1:N).

Intent Recognition

Combines STT with CLU to extract intent and entities from speech in one step.

Custom Neural Voice

Train a custom TTS voice model from voice samples. Requires Limited Access approval.

Speech SDK Quickstart Pattern

import azure.cognitiveservices.speech as speechsdk

config = speechsdk.SpeechConfig(subscription=key, region=region)
config.speech_recognition_language = "en-US"

# From microphone
audio_config = speechsdk.AudioConfig(use_default_microphone=True)
recognizer = speechsdk.SpeechRecognizer(speech_config=config, audio_config=audio_config)
result = recognizer.recognize_once_async().get()

if result.reason == speechsdk.ResultReason.RecognizedSpeech:
    print(f"Recognized: {result.text}")

SSML — Speech Synthesis Markup Language

SSML lets you control pronunciation, speaking rate, pitch, pauses, and voice selection in TTS.

<speak version="1.0" xmlns="http://www.w3.org/2001/10/synthesis" xml:lang="en-US">
  <voice name="en-US-JennyNeural">
    <prosody rate="slow" pitch="+5%">
      Welcome to <emphasis level="strong">Azure AI</emphasis> Speech services.
    </prosody>
    <break time="500ms"/>
    How can I help you today?
  </voice>
</speak>

Custom Speech

Improve STT accuracy for domain-specific terms, accents, and environments by providing custom training data:

  • Audio + Human-labeled transcripts — most accurate adaptation, needs 1–20 hrs audio
  • Plain text — improves language model (words, phrases, n-grams)
  • Pronunciation — add custom pronunciations for unusual words

Batch Transcription

Transcribe large audio files asynchronously. Submit via REST API, audio must be in Azure Blob Storage.

POST /speechtotext/v3.1/transcriptions
{
  "contentUrls": ["https://storage.blob.core.windows.net/.../audio.wav"],
  "locale": "en-US",
  "displayName": "My batch transcription"
}
Exam Tip — Storage for Batch Transcription (Q22) Audio files for batch transcription must be stored in Azure Blob Storage. The API accepts SAS URLs or managed identity access to Blob containers directly. Results are also written back to a Blob location you specify. Supported formats: WAV, MP3, OGG, FLAC.

Speaker Recognition & Voice Profiles

Speaker Recognition identifies or verifies who is speaking by analysing voice characteristics. It relies on voice profiles created from enrolled audio.

Profile TypeUse CaseHow It Works
TextDependentVerification Speaker verification (1:1) — is this the enrolled person? Speaker says a specific passphrase during enrolment and at verification time
TextIndependentIdentification Speaker identification (1:N) and diarization — who is speaking at each moment? Speaker can say anything; no passphrase required. Works on natural speech.
Exam Tip — Voice Profile for Meeting Recordings (Q14) To identify who is speaking at each point in a meeting recording, use the TextIndependentIdentification voice profile type. It does not require a passphrase and supports 1:N matching across multiple speakers (diarization). TextDependentVerification is for 1:1 passphrase-based verification only.

Pattern Matching Intent Recognition

Pattern matching is a lightweight, offline intent recognition approach — no cloud call needed after the SDK is downloaded. It matches utterances against defined patterns rather than an ML model.

ExactPhrase

Matches a specific phrase exactly from a defined catalog. Use when you have a fixed list of phrases to detect as entities.

Any

Matches any text in a position — a wildcard slot within a pattern.

IntegerRange / FloatRange

Matches numbers within a specified range (e.g., quantities, temperatures).

Exam Tip — Pattern Matching Entities (Q44) To detect entities defined in a catalog of phrases using pattern matching intent recognition, use the ExactPhrase entity type. It performs exact string matching against your phrase list with no ML inference.

Speech Recognition in Noisy Environments

For noisy environments, recommend the Whisper model (available via Azure AI Speech and Azure OpenAI). Whisper is trained on diverse, real-world audio and handles:

  • Background noise and ambient sound
  • Multiple accents and dialects
  • Multi-language audio

Alternatively, use Custom Speech with acoustically-matched training data recorded in the target noisy environment to improve accuracy.
