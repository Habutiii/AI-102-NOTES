Language Understanding — CLU (Conversational Language Understanding)

Key Takeaways:
  • CLU successor to LUIS: understands intent and extracts entities from utterances
  • Build models using the Language Studio portal or SDK
  • Train, evaluate, and deploy CLU models to prediction endpoints
  • Support custom entity types with machine-learned and list-based entities
  • Integrate with Azure Bot Service and Azure OpenAI for conversational AI

CLU Overview

CLU (successor to LUIS) understands the intent behind natural language utterances and extracts entities. Used in chatbots, voice commands, and virtual assistants.

Intent

The goal or purpose of a user's utterance. Example: "Book a flight to Paris" → intent = BookFlight. Always include a None intent for utterances outside the app's scope.

Entity

Key information extracted from the utterance. Example: "Paris" → Destination entity. Types: Machine-learned, List, Regex, Prebuilt.

Utterance

A sample user input used to train the model. More diverse utterances per intent = better model. Minimum ~5 per intent; aim for 15–30+.

Entity Types

Entity TypeWhen to UseExample
Machine-learnedFlexible entities from contextProduct names, descriptions
ListFixed set of exact matchesCities: ["New York", "London"]
RegexPattern-based matchingFlight numbers: [A-Z]{2}\d{4}
PrebuiltCommon types (numbers, dates, etc.)datetimeV2, number, email, currency

CLU Development Workflow

Create Language resource → Create CLU project in Language Studio
→ Define intents → Add utterances → Label entities
→ Train → Evaluate (precision/recall/F1) → Deploy → Call prediction API

Evaluation Metrics

After training, CLU reports three metrics per intent and entity. Understanding the formulas is exam-tested.

Confusion Matrix Terms

TermMeaning
True Positive (TP)Model predicted the label — and was correct
True Negative (TN)Model did not predict the label — and was correct (it was indeed absent)
False Positive (FP)Model predicted the label — but was wrong (it wasn't actually there)
False Negative (FN)Model did not predict the label — but was wrong (it was actually there, and was missed)
MetricFormulaMeaningPrioritise when…
Precision TP ÷ (TP + FP) Of everything the model labelled as X, how many were actually X? High precision = few false positives. False positives are costly (e.g., triggering the wrong action)
Recall TP ÷ (TP + FN) Of all actual X instances, how many did the model find? High recall = few missed positives. Missing a true positive is costly (e.g., safety alerts)
F1 Score 2 × (P × R) ÷ (P + R) Harmonic mean of Precision and Recall. Balanced measure. Neither false positives nor false negatives can be tolerated
Exam Tip — Precision Definition (Q32) "Ratio between correctly identified positives (true positives) and all identified positives" = Precision = TP ÷ (TP + FP). Recall divides by (TP + FN) — i.e., all actual positives, not all identified ones.

Orchestration Workflow

An Orchestration project routes user input to multiple CLU projects or Q&A knowledge bases. Acts as a dispatcher — ideal for complex bots with multiple domains.

Use case A single bot that can answer HR questions (Q&A KB), book meetings (CLU project A), and order supplies (CLU project B). The Orchestration project picks the right backend.

What you CAN do in an Orchestration Workflow (Q20)

  • Route utterances to the correct CLU model or Q&A project based on intent classification
  • Connect multiple language understanding projects (CLU + Custom Q&A) within one unified endpoint

What you CANNOT do

  • Train new intents directly inside the orchestration workflow — it only routes to existing projects
  • Perform entity extraction — that happens in the downstream CLU projects

Bot Framework Integration

CLU integrates with Azure Bot Service and Bot Framework SDK. The bot middleware calls the CLU prediction endpoint, then routes based on the top intent.

  • DirectLine channel — REST-based bot connection for custom clients
  • Channels — Teams, Webchat, Slack, Facebook, etc.
  • Bot Service — hosts and manages the bot; provides channel routing
