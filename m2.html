Azure AI Vision (Image Analysis)

The Azure AI Vision service (v4.0 Image Analysis API) provides a unified endpoint for analysing images. It replaces earlier Computer Vision v3 endpoints.

Caption & Dense Captions

Generates human-readable description of an image and per-region captions. Uses Florence foundation model.

Object Detection

Detects objects and returns bounding boxes with labels and confidence scores.

Smart Crop

Returns cropping coordinates that preserve the most visually interesting region.

Tags

Returns list of content tags with confidence scores (e.g., "outdoor", "sky", "person").

People Detection

Detects people in an image (bounding boxes). Does NOT identify individuals.

Background Removal

Segments the subject from background. Returns PNG with alpha channel.

API Call (v4.0)

POST https://<endpoint>/computervision/imageanalysis:analyze
    ?api-version=2023-02-01-preview
    &features=Caption,Tags,Objects
    &language=en

Content-Type: application/json
Ocp-Apim-Subscription-Key: <key>

{ "url": "https://example.com/image.jpg" }

Two options exist for reading text:

# Submit Read operation
POST /computervision/imageanalysis:analyze?features=Read

# Python SDK example
from azure.ai.vision.imageanalysis import ImageAnalysisClient
from azure.core.credentials import AzureKeyCredential

client = ImageAnalysisClient(endpoint, AzureKeyCredential(key))
result = client.analyze(image_url=url, visual_features=[VisualFeatures.READ])
for line in result.read.blocks[0].lines:
    print(line.text)

Custom Vision

Azure AI Custom Vision lets you train models for image classification and object detection on your own dataset without coding.

Image Classification

Assigns one or more labels to an entire image. Types: Multiclass (one label) or Multilabel (multiple labels). Min: 5 images per tag to train.

Object Detection

Identifies object locations with bounding boxes. Min: 15 images per tag. Labels must be drawn per image.

Retirement Notice — September 30, 2026 Azure AI Custom Vision is being retired. For new projects, use Azure AI Vision custom models (image classification and object detection fine-tuned via Vision Studio) or Florence-based model fine-tuning in Azure AI Foundry. Custom Vision content remains on the current AI-102 exam.
Exam Tip Custom Vision uses two separate resources: a Training resource and a Prediction resource. Training key is used to create/train; Prediction key is used for inference calls. You export models as ONNX, TensorFlow, CoreML, or Docker container for offline use.
ConceptDetail
IterationEach training run creates an iteration. Publish an iteration to make it available for prediction.
DomainCompact domains (faster, exportable) vs. General domains (more accurate, cloud only)

Performance metricsPrecision, Recall, AP (Average Precision) per tag

Face API

The Face API detects, analyzes, and compares human faces.

Responsible AI Note Face identification, verification, and emotion detection features require Limited Access approval from Microsoft. Default capability is face detection only (locations, attributes).
Face Detection

Returns face rectangles, age estimate, gender, emotion, accessories, blur, exposure, noise, occlusion, headPose, smile, glasses.

Face Verification

Compares two faces — are they the same person? Returns confidence 0–1.

Face Identification

Matches a face against a PersonGroup of known individuals. Requires training a PersonGroup first.

Find Similar

Finds similar faces from a FaceList without a PersonGroup.

Face Identification Workflow

1. Create PersonGroup → 2. Add Persons → 3. Add Face images → 4. Train PersonGroup
5. Detect face in new image (get faceId) → 6. Identify faceId against PersonGroup

Extracts insights from video: transcripts, speaker identification, face detection, keyframes, labels, sentiments, brands, named entities, scene changes, and more.

Multilingual Identification

To enable automatic language detection per segment, set the sourceLanguage parameter to "multi-language" when uploading a video. This tells the indexer to detect languages per segment rather than using a single fixed language.

Custom Language Model

Upload a plain-text file of domain-specific words and phrases to improve speech-recognition accuracy for specialised vocabulary.

Custom Language Model — Upload Rules
  • Maximum file size: 2 GB
  • Encoding: UTF-8
  • One entry per line
  • Lines containing special characters (& < > " #) are silently discarded — remove or replace them before upload
Exam Tip (Q19) — Custom Language Model Best Practices Use plain text files, include domain-specific vocabulary, and avoid special characters in entries. Any line with a special character is dropped by the indexer without warning.

Custom People Model

Train a Custom People Model by uploading images of specific individuals. Once trained, Video Indexer can recognise those people in videos — extending beyond the built-in celebrity recognition capability.

Custom Vision — Additional Exam Topics

Performance Assessment After Training

After training an iteration, evaluate it using the Performance tab in the Custom Vision portal. Key metrics are:

MetricMeaning
PrecisionOf all predictions made, what fraction were correct?
RecallOf all actual positives, what fraction did the model find?
AP (Average Precision)Area under the precision-recall curve — overall model quality per tag

Use Quick Test (also on the Performance tab) to test with a new image without making a prediction endpoint call.

Publishing Iterations

A trained iteration must be published to a Prediction resource before it can be used for inference via the API. Unpublished iterations can only be tested using Quick Test inside the portal.

Exam Tip (Q26) — Publishing Custom Vision has separate Training and Prediction resources. Publishing an iteration links it to a Prediction resource endpoint. Without publishing, external applications cannot call the model.

Firewall / Network Access

When firewall rules are enabled on an Azure AI Services account, only requests from allowed IPs or VNets are accepted. For internet-hosted applications, add the application's outbound IP address to the allowed list on the resource's Networking tab in the Azure portal.

Network Access (Q25) If your app receives HTTP 403 or connection errors after enabling firewall rules, check that the app's public outbound IP is listed under Firewall and virtual networks on the Azure AI resource.

Image Analysis — Additional Exam Topics

Color Analysis (Q34)

The Image Analysis API can return color information about an image:

Dominant Foreground Color

The most prominent color in the foreground subject. Returns a basic color name.

Dominant Background Color

The most prominent color in the background. Returns a basic color name.

Accent Color

The single most vibrant color in the image, returned as a hex code.

Background and foreground color values are limited to these basic color names: Black, Blue, Brown, Gray, Green, Orange, Pink, Purple, Red, Teal, White, Yellow.

Exam Tip (Q16) If a question asks how to find where an object is in an image, choose the Objects visual feature, not Tags. Only Objects returns bounding box coordinates.

People Detection in Video (Q57)

Use the People visual feature (also referred to as spatialAnalysis) in Azure AI Vision to detect the presence of people in a video feed. The API returns a bounding box for each detected person per frame. This is distinct from Face API identification — it detects presence, not identity.
